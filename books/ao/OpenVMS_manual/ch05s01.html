Operating In MUPIP

The default installation procedure places the MUPIP utility program in a directory specified by the logical name GTM$DIST

The installation also creates command files that define GTM$DIST and assigns the symbol MUPIP to be the foreign command, $GTM$DIST:MUPI*P. The asterisk (*) permits abbreviation of the symbol to four (4) characters.

Invoke MUPIP using the GT.M symbol MUPIP following command at the DCL prompt. If this does not work, consult your system manager to investigate setup and file access issues.

$ MUPIP
MUPIP>

When MUPIP is ready to accept commands, the system displays the MUPIP> prompt. Enter the EXIT command at the MUPIP> prompt to stop the utility. MUPIP is designed to perform one operation at a time, and automatically terminates after most operations.

When additional information appears on the DCL command line after the MUPIP command, MUPIP processes the additional information as its command. Since MUPIP performs one operation at a time, FIS recommends using this form of invocation for MUPIP operations. 

$ MUPIP HELP

Use of Global Directories

The logical name GTM$GBLDIR specifies the Global Directory. Define GTM$GBLDIR using the DCL command DEFINE. The system manager may define GTM$GBLDIR in a GROUP or SYSTEM table. Individual users define GTM$GBLDIR in their LOGIN.COM file or other command files.

$ DEFINE GTM$GBLDIR PROD.GLD

For more information on the Global Directory, refer to the "Global Directory Editor" chapter.

For more information on defining logical names, refer to the OpenVMS User's Manual.
