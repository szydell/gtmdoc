Shutdown

Stopping GT.M involves ensuring that all users have terminated their access to the database files. This termination serves as a critical element in maintaining database integrity. For performance reasons, GT.M caches data in memory while the system is running. When all users have properly exited from GT.M, the system writes all global data from the memory cache to disk. Without proper termination, the data in the cache is not written to the disk and is lost when VMS shuts down. Run the GTMSTOP.COM procedure to stop all GT.M users and properly terminate GT.M. Place a copy or invocation of GTMSTOP in SYS$MANAGER:SHUTDWN.COM.

In the event of abnormal termination of the GT.M process, you must execute a MUPIP RUNDOWN. Refer to the "MUPIP" chapter in GT.M Administration and Operations Guide for details.
