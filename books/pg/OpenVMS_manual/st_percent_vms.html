%ST

The %ST utility displays the number of current users of the GT.M run-time library and information about those processes that you have privileges to view. You must have the VMS SYSLCK privilege to run this utility. If there are no users accessing GT.M, the message "No current GT.M users" appears.

%ST displays the access method used by the processes. For a list of the possible access methods, refer to the Access Methods table in GT.M Programmer's Guide.

Access Methods

Access Method

Description

Detach

Process is no longer connected

Network

Process logged on via a DECnet task

Batch

Process logged on via a batch job

Local

Process logged on via a local terminal

Dialup

Process logged on via a dialup terminal

Remote

Process logged in via a network
