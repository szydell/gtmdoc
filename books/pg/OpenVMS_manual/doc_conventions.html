Document Conventions

This chapter uses the following mnemonics to describe when a deviceparameter applies:

TRM Valid for terminals

S/S: Valid for sequential files when used with SPOOL or SUBMIT

SUB: Valid for sequential files when used with SUBMIT

SPL: Valid for sequential files when used with SPOOL

SD: Valid for sequential disk files independent of SPOOL or SUBMIT

MBX: Valid for mailboxes

NULL: Valid for null devices

SOC: Valid for socket devices

Many of the deviceparameter defaults shown are the basic VMS defaults, and are subject to modification before the invocation of GT.M. Modify terminal characteristics with the DCL SET TERMINAL command. Modify disk file characteristics with the DCL INITIALIZE, SET and MOUNT commands.
