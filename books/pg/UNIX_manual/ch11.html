  • In “Using External Calls”, Add null address and negative length handling
Revision V6.3-006 26 October 2018

  • In “Using External Calls”, added SIGSAFE keyword

  • In “Pre-allocation of Output Parameters”, specify that when an external call exceeds its specified preallocation (gtm_string_t * or gtm_char_t * output), GT.M produces an EXCEEDSPREALLOC error.

  • In “Rules to Follow in Call-Ins”, specify that using gtm_malloc is mandatory to allocate returns of pointer types to prevent memory leaks.

Revision V6.3-005 29 June 2018

  • In “Nested Call-Ins”, tweak ZGOTO description for GTMCI behavior
Revision V6.3-001 20 March 2017

  • In “Building Standalone Programs”, removed instructions for platforms that are no longer supported.

  • In “Creating a Shareable Library”, removed examples for currently unsupported platforms.

  • In “Using External Calls”, removed Tru64 and fixed formatting

  • In “gtmxc_types.h”, removed Tru64

  • In “Relevant files for Call-Ins”, removed HP-UX
Revision V6.1-000 28 August 2014

  • In “Access to Non-M Routines”, added information about the external call arguments that do not specify a value and corrected the examples.

  • In “Using External Calls”, added a note about how arguments that do not specify a value translate to default values in C.

Revision V6.0-003 24 February 2014

  • Improved the description of the “Pre-allocation of Output Parameters” section.

  • In “Rules to Follow in Call-Ins”, added a recommendation to not mix GT.M device input using read() system service with bufferred input services in fgets() family.

  • Added a section called “Type Limits for Call-ins and Call-outs”.
