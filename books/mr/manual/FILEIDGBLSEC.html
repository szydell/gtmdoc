FILEIDGBLSEC

FILEIDGBLSEC, File ID in global section does not match with the database file

Run Time Error: When a GT.M process attaches to a database and finds the corresponding shared memory structures initialized already, it performs integrity checks on the shared memory contents to ensure that they correspond back to the database file. When the shared memory copy of the database file ID does not match with the actual file ID of the database, this error is issued.

Action: Perform a MUPIP RUNDOWN on that region. If it fails with the same FILEIDGBLSEC error, then the shared memory contents are corrupt. Consult your GT.M support channel before proceeding further.
